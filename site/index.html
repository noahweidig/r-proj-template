FAQ

    

      

        
Which packages handle reading and writing data?

        
The core set is tidyverse-first. vroom is included for fast, tidy reading and writing of delimited files (it returns a tibble and plays nicely with dplyr), and conflicted can be toggled on to make any ambiguous function calls explicit.

      

      

        
Makefile, run.R, or targets — which do I need?

        
make all and Rscript run.R always re-run every step. targets::tar_make() builds a dependency graph and only reruns steps whose inputs changed — pick it for anything beyond a quick script.

      

      

        
renv, Docker, or Dev Containers?

        
They stack, not compete: renv pins R packages, Docker pins the OS + R version too, and a Dev Container wraps Docker with a ready-to-code VS Code/Codespaces setup. Use whichever matches how much environment drift you're worried about.

      

      

        
Is anything ever uploaded?

        
No. The whole builder — form, preview, zip generation — runs client-side in your browser with JSZip. Export/share features write to your clipboard, disk, or the URL bar only.
